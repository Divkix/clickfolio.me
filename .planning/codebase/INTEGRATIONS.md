# External Integrations

**Analysis Date:** 2026-02-26

## APIs & External Services

**Authentication:**
- Google OAuth 2.0 (via Better Auth)
  - SDK/Client: `better-auth` with built-in Google strategy
  - Env vars: `GOOGLE_CLIENT_ID`, `GOOGLE_CLIENT_SECRET`
  - Scope: email, profile
  - Implementation: `lib/auth/index.ts`

**AI Resume Parsing:**
- OpenRouter (via Cloudflare AI Gateway)
  - SDK/Client: `@ai-sdk/openai-compatible` (Vercel AI SDK)
  - Model: `openai/gpt-oss-120b:nitro` (default, configurable via `AI_MODEL` env var)
  - Provider routing: Prefers Cerebras (fp16 ~694tps) and DeepInfra (bf16 ~228tps)
  - Features: Structured JSON schema output with response healing plugin
  - Fallback: Text-based parsing with JSON repair if structured output fails
  - Implementation: `lib/ai/ai-parser.ts`, `lib/ai/pdf-extract.ts`
  - Env vars: `CF_AI_GATEWAY_ACCOUNT_ID`, `CF_AI_GATEWAY_ID`, `CF_AIG_AUTH_TOKEN`

**Email:**
- Resend (transactional emails)
  - SDK/Client: HTTP API via `ofetch` (lightweight, custom client in `lib/email/resend.ts`)
  - Usage: Password reset, email verification
  - Endpoint: `https://api.resend.com/emails`
  - Auth: Bearer token in Authorization header
  - From domain: Derived from `BETTER_AUTH_URL` hostname (e.g., noreply@clickfolio.me)
  - Test domain: onboarding@resend.dev (for localhost)
  - Env var: `RESEND_API_KEY` (optional, 3k/month free tier)
  - Implementation: `lib/email/resend.ts`

**Analytics:**
- Umami (self-hosted at analytics.divkix.me)
  - API: REST API v2.12+ compatible
  - Auth: JWT token via POST /api/auth/login (1-hour TTL, auto-refreshes)
  - Usage: Dashboard stats, portfolio view tracking, referral click attribution
  - Endpoints:
    - `/api/auth/login` - Get JWT token
    - `/api/websites/{website_id}/stats` - Pageviews, visitors, bounce rate
    - `/api/websites/{website_id}/pageviews` - Time-series pageviews and sessions
    - `/api/websites/{website_id}/metrics/{metric}` - Specific metric query
  - Env vars: `NEXT_PUBLIC_UMAMI_WEBSITE_ID` (public), `UMAMI_API_URL`, `UMAMI_USERNAME`, `UMAMI_PASSWORD`
  - Implementation: `lib/umami/client.ts`
  - Token caching: Module-level cache (60 min TTL, auto-clear on 401)

**CDN & Cache:**
- Cloudflare Cache Purge API
  - Usage: Purge edge cache on privacy-sensitive changes (hide phone, address, etc.)
  - Endpoint: `https://api.cloudflare.com/client/v4/zones/{zone_id}/purge_cache`
  - Auth: Bearer token
  - Env vars: `CF_ZONE_ID`, `CF_CACHE_PURGE_API_TOKEN`
  - Implementation: `lib/cloudflare-cache-purge.ts`
  - Best-effort: Logs errors, never throws

## Data Storage

**Databases:**
- D1 (Cloudflare SQLite)
  - Connection: Binding `DB` in `wrangler.jsonc`
  - Client: Drizzle ORM (`drizzle-orm/d1`)
  - Schema: `lib/db/schema.ts` (10 tables: user, session, account, verification, resumes, siteData, handleChanges, uploadRateLimits, referralClicks, and Better Auth tables)
  - Features: Full-text search not used, JSON stored as TEXT (parsed on retrieval)
  - Migrations: Generated by Drizzle Kit, applied via Wrangler CLI
  - Implementation: `lib/db/index.ts`, `lib/db/session.ts`

**File Storage:**
- R2 (Cloudflare object storage)
  - Binding: `CLICKFOLIO_R2_BUCKET` (clickfolio-bucket)
  - Stored artifacts: PDF uploads (temp + claimed), parsed resume content (JSON), OG images
  - Access: Direct binding via `lib/r2.ts` wrapper (no S3 SDK, no credentials needed)
  - CORS: Configured for localhost:3000 and clickfolio.me
  - Client library: Custom wrapper in `lib/r2.ts` (get, put, delete, head, copy, list)

**Caching:**
- Cloudflare KV (key-value store)
  - Binding: `DISPOSABLE_DOMAINS` namespace
  - Usage: Blocklist of disposable email domains (synced via cron)
  - Syncing: Daily cron job (`lib/cron/sync-disposable-domains.ts`)
  - Implementation: `lib/email/disposable-check.ts`

## Authentication & Identity

**Auth Provider:**
- Better Auth (self-hosted on app)
  - Implementation: `lib/auth/index.ts` (server), `lib/auth/client.ts` (client hooks)
  - Strategies: Google OAuth, email/password, email verification, password reset
  - Database: D1 via Drizzle adapter
  - Tables: user, session, account, verification (auto-managed by Better Auth)
  - Custom fields on user: handle, headline, privacySettings, onboardingCompleted, role, roleSource, referredBy, referredAt, isPro, referralCode, referralCount, isAdmin, showInDirectory
  - Session validation:
    - Middleware (`lib/auth/middleware.ts`): Checks session cookie existence (edge, no DB access)

  - Password reset: Via Resend email, 1-hour token expiry
  - Email verification: Triggered on signup, 24-hour token expiry

## Webhooks & Callbacks

**Incoming Webhooks:**
- Resume status notifications (internal)
  - Endpoint: `POST /ws/resume-status?resume_id=X` (WebSocket upgrade, not HTTP webhook)
  - Handler: Durable Object `ResumeStatusDO` (keyed by resumeId)
  - Purpose: Real-time status updates via WebSocket, triggered by queue consumer POST /notify
  - Implementation: `lib/durable-objects/resume-status.ts`

**Outgoing Webhooks:**
- DLQ alerting (optional)
  - Endpoint: Configured via `ALERT_WEBHOOK_URL` (Slack/Discord webhook format)
  - Trigger: Resume parsing failures (retries exhausted)
  - Data: Error message, resume ID, user email
  - Implementation: `lib/queue/dlq-consumer.ts`

**Cloudflare Queues (internal async messaging):**
- Queue: `resume-parse-queue` (main parsing queue)
  - Producer: API route `/api/resume/claim` (triggers queue when user claims upload)
  - Consumer: `worker.ts` queue handler (processes 1 message at a time, max 3 retries)
  - Payload: `ResumeParseMessage` (resumeId, userId, uploadKey, attemptCount)
  - Processing: Extracts PDF text → AI parse → normalizes → writes to siteData
  - Implementation: `lib/queue/consumer.ts`

- Queue: `resume-parse-dlq` (dead letter queue)
  - Consumer: `worker.ts` DLQ handler (logs errors, posts to webhook if configured)
  - Max retries: 0 (no retry on DLQ)
  - Classification: Retryable errors (timeout, rate limit) trigger main queue retry; non-retryable (schema, empty text) go straight to DLQ
  - Implementation: `lib/queue/dlq-consumer.ts`, `lib/queue/errors.ts`

**Cloudflare Cron Triggers:**
- `0 3 * * *` (3 AM UTC) → `GET /api/cron/cleanup` - Session/verification expiry cleanup
- `0 4 * * *` (4 AM UTC) → Sync disposable email domain blocklist from upstream
- `*/15 * * * *` (every 15 min) → `GET /api/cron/recover-orphaned` - Recover orphaned resumes (claimed but not parsed)
- Auth: `CRON_SECRET` Bearer token in Authorization header (validates at route handler)
- Implementation: `lib/cron/cleanup.ts`, `lib/cron/sync-disposable-domains.ts`, `lib/cron/recover-orphaned.ts`

## Environment Configuration

**Required env vars (no defaults):**
- `BETTER_AUTH_SECRET` - Session signing key (32+ chars recommended)
- `BETTER_AUTH_URL` - App URL (used for session cookies, email domains, OAuth callback)
- `GOOGLE_CLIENT_ID`, `GOOGLE_CLIENT_SECRET` - Google OAuth credentials
- `CF_AI_GATEWAY_ACCOUNT_ID`, `CF_AI_GATEWAY_ID`, `CF_AIG_AUTH_TOKEN` - Cloudflare AI Gateway

**Optional env vars (with fallbacks):**
- `RESEND_API_KEY` - Resend email API (skips email sending if absent)
- `CF_ZONE_ID`, `CF_CACHE_PURGE_API_TOKEN` - Cache purge (silent fail if absent)
- `UMAMI_API_URL`, `UMAMI_USERNAME`, `UMAMI_PASSWORD` - Analytics (falls back to read-only if absent)
- `CRON_SECRET` - Cron auth (validates all three cron endpoints)
- `ALERT_CHANNEL`, `ALERT_WEBHOOK_URL` - DLQ alerting (defaults to logpush)

**Public env vars:**
- `NEXT_PUBLIC_UMAMI_WEBSITE_ID` - Umami tracker ID (included in client bundle)
- `NODE_ENV` - "production" (set in wrangler.jsonc)
- `AI_MODEL` - LLM model ID (default: "openai/gpt-oss-120b:nitro")

**Secrets location:**
- Dev: `.dev.vars` (Vite/Wrangler loads)
- Prod: Cloudflare secrets (`wrangler secret put KEY_NAME`)
- Never commit: `.env*` files with actual values
